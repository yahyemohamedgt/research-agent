# verbatim_phrases must exclude paid sources — enforced in code, same pattern as the Twitter/X split

Once the Foreplay query fix (ADR 0009) started returning real ads, Sonnet began occasionally citing a Foreplay ad's hook text as an organic `verbatim_phrases` entry — in one observed case labeling it `"Instagram (Ad: brand)"` itself, so the model clearly knew it was an ad and cited it anyway. The hallucination checker passed it as `VERIFIED`, because `eval.py`'s corpus includes `foreplay_ads` hook text, so a paid quote always "verifies" against itself — the eval layer has no way to distinguish organic content from paid content within the corpus it checks against. This is the identical failure mode that produced the Twitter/X → `x_signals` split: a non-organic source leaking into a field whose entire premise is "this is real community language." The system prompt already says `verbatim_phrases` should be organic-only, but the prompt is a hint, not a guarantee (same lesson as ADR 0008). The fix filters `synthesize()`'s output in code: any `verbatim_phrases` entry whose URL matches a collected Foreplay ad's URL is dropped before the Twitter-URL filter runs. The tell that made this catchable at all was the URL shape — a product/landing page, not a native platform post URL — which is the same kind of code-checkable signal used for `_is_twitter_url`.

Related, same principle: Instagram's raw API response includes `is_ad`/`is_paid_partnership` flags that `collect_instagram` now filters out at collection time, so a sponsored Instagram post can't enter the organic corpus in the first place. Two different enforcement points (collector-level filtering for Instagram, post-synthesis filtering for Foreplay) because the two platforms surface the "this is paid" signal at different stages — Instagram tags it on the raw post, Foreplay ads only reveal themselves as paid by not being a native post URL once cited.
