# Foreplay query must name a concrete product/category, not a marketing-strategy sentence

The plan node's `foreplay_query` field was originally instructed to be "one level broader than the Reddit query — wider vertical, broader audience framing." In practice this produced abstract sentences like "effective marketing strategies for women's health and beauty products advertising hooks," which Foreplay's ad-discovery search — built to match ad creative content, not commentary about marketing — couldn't resolve to anything real. Every skincare/DTC test run reported a false "first mover opportunity" as a result, even though 25+ real competitors were running live ads in that category. Verified live: concrete terms like `"collagen supplement"` and `"nurse scheduling software"` return dozens of real, on-topic ads from the same API; the "broader framing" style consistently returned 0–1 irrelevant results. The fix is a schema description and prompt instruction that explicitly asks for a product/category noun phrase, with examples, and explicitly rules out marketing-strategy language.
