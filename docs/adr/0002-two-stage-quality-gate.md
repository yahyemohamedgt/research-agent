# Two-stage quality gate: Python volume floor before Sonnet relevance check

The quality gate runs two checks in sequence. First: a Python count — if combined collector results are below 10, the run fails immediately with `failed_at="volume"`, no LLM call made. Second: a Sonnet call that reads the collected content and judges whether it actually addresses the research question for the stated audience — fails with `failed_at="relevance"` if not. The two stages are deliberately separate because they diagnose different problems: volume failure means the queries were wrong or the topic is too niche; relevance failure means there is content out there but the agent found the wrong angle. The caller gets a different `suggestion` for each case. Running Sonnet first on thin data would waste a call and produce an unreliable judgment.
