# Research Agent

A LangGraph agent that takes an audience description and research question, collects organic and paid signal across 10 platforms (Reddit, Exa, YouTube, Foreplay, Twitter/X, TikTok, Instagram, Threads, HN, GitHub), and returns a structured brief telling a media buyer whether the community confirms the premise, what language they use to talk about it, and what the paid-ad landscape looks like.

## Language

### Inputs

**Run**:
A single invocation of `run_agent(audience, question)`. Returns either a Brief or an AgentError.
_Avoid_: job, task, query, request

**Audience description**:
Free-text string describing who the agent should find signal about — demographics, context, platform behaviour. Passed directly to the plan node.
_Avoid_: target, persona, segment

**Research question**:
Free-text string describing what the agent should find out about that audience. Paired with the audience description throughout the graph.
_Avoid_: topic, query, prompt

### Graph internals

**Query plan**:
Structured per-platform search parameters generated by the plan node. Contains queries for all 10 platforms: subreddits, Reddit search string, Exa semantic query, YouTube terms, Foreplay query, Twitter/X query, TikTok query, Instagram query, Threads query, HN query, GitHub query. The only thing the collectors read — they execute what the plan hands them.
_Avoid_: search config, parameters, settings

**Subreddit principle**:
The plan node always prefers identity/community-specific subreddits over broad topic ones (e.g. a Muslim audience → r/MuslimLounge, not r/religion). Applied on every plan and replan pass.

**Broadening note**:
Optional field inside a query plan, written by the plan node on retry passes. Records what was tried and why it failed so the next plan generates meaningfully wider queries. The plan node's memory across retries.
_Avoid_: retry hint, failure context

**Volume floor**:
The first stage of the quality gate. Python check — no LLM call — that fails the run immediately if combined collector results fall below 10. Distinguishes "not enough data" from "wrong data".
_Avoid_: result count, minimum threshold

**Relevance check**:
The second stage of the quality gate. Sonnet call that evaluates whether the collected signal actually addresses the research question for the stated audience. Only runs if the volume floor passes.
_Avoid_: quality check, LLM gate, relevance score

### Outputs

**Brief**:
The structured output of a successful run. Contains dominant emotion, exactly five verbatim phrases, up to five X signals, a GREEN or RED signal judgment with reasoning, a decision summary, winning ads and paid-competition read, platform breakdown, and cited sources.
_Avoid_: report, result, response, output

**Signal**:
The GREEN/RED judgment in a Brief about whether the collected content confirms the research premise for the stated audience. GREEN means the community is loud, consistent, and on-topic. RED means content came back but doesn't confirm the premise — off-audience, thin, or contradictory. Distinct from a quality gate failure, which means not enough content came back at all.
_Avoid_: confidence, score, verdict, result

**Verbatim phrases**:
Exactly five direct quotes pulled from the collected corpus, sourced only from Reddit, Exa, YouTube, Foreplay, TikTok, Instagram, Threads, HN, or GitHub — never Twitter/X. The actual language the community uses. Fixed at five — enough to see a pattern, predictable for the delivery layer.
_Avoid_: quotes, examples, samples, highlights, tweets

**X signals**:
Up to five Twitter/X items in a Brief, Grok-synthesized rather than verbatim. Kept separate from verbatim phrases because Grok summarizes tweet text even under a verbatim instruction, making Twitter content unverifiable against the corpus — see ADR 0008. Honestly labeled as synthesized, not quoted.
_Avoid_: tweets, verbatim phrases, quotes

**Winning ads**:
Paid creative pulled from Foreplay, each with brand, hook text, emotional driver, days running, and URL. Used to judge paid competition and paid gap.
_Avoid_: top ads, ad examples

**Paid competition** / **Paid gap**:
Paid competition is a HIGH/MEDIUM/LOW/NONE read on how saturated the paid-ad space is for this audience/question. Paid gap is the plain-English note on what the winning ads are missing — the opening a media buyer could exploit.
_Avoid_: ad saturation, competitive analysis

**Decision summary**:
One paragraph of plain English in a Brief, written for someone who will not read the verbatim phrases. The field that gets read on WhatsApp when the brief arrives. Everything else in the Brief is supporting detail.
_Avoid_: summary, TL;DR, overview

**AgentError**:
Structured return from a failed run — not a raised exception. Contains the failure reason, what was searched across all retry passes, the retry count, a suggestion for the caller, and which gate stage killed the run (volume or relevance).
_Avoid_: exception, error, failure response

### Eval layer

**Hallucination check**:
Python check run only against `verbatim_phrases` (never `x_signals`, since those are synthesized by design). Confirms each phrase appears in the collected corpus via substring match or an 8-word sliding-window fuzzy fallback.
_Avoid_: fact check, verification

**Judge score**:
GPT-4o-mini's 1–10 rating of a Brief across specificity, evidence, actionability, and hook quality — a cross-model check, deliberately not the same model that wrote the Brief.
_Avoid_: quality score, LLM grade

**Eval passed**:
True only if the hallucination check found zero violations and the judge's overall score is ≥ 6. Distinct from the quality gate, which runs before synthesis; eval runs after.
_Avoid_: passed, validated
